# JSON Artifact Inventory

## Purpose

This document classifies the JSON and JSONL artifacts currently present under
`data/` by operational importance.

The goal is to answer four questions:
- is this file actively consumed by code?
- is it generated during active parsing or search workflows?
- is it only a report/export artifact?
- can it be archived or deleted without breaking normal operation?

## Summary

Most JSON files in this repo are not core runtime dependencies.

They fall into four groups:

1. active workflow state
2. active legacy ingest interchange files
3. generated reports and analysis outputs
4. ad hoc research or scratch artifacts

The database and source PDFs are the real durable system of record. Many JSON
files are intermediate outputs or analyst-side convenience exports.
